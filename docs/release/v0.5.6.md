# Munin v0.5.6

Munin v0.5.6 moves live runtime context ownership fully into Munin and fixes the
Session Brain output contract.

## Highlights

- `munin brain` now emits compiled public context only. It no longer exposes raw
  transcript paths, raw message blocks, repeated clipped subgoals, or stale
  progress chatter.
- Partial strategy memory is accepted when a formal strategy kernel is not yet
  complete, so project focus, KPIs, and business strategy can still surface from
  compiled Memory OS state.
- Claude and Codex integration assets now ship from Munin's versioned asset
  bundle, including the `munin-runtime.js` Claude hook.
- Munin-owned data/config/artifact roots are now the default runtime path.
- Release packaging includes the new runtime context module and release-check
  script.

## Verification

- `cargo test`
- `cargo build --release`
- `cargo run --locked --bin munin -- install --check-resolvable`
- `cargo package`
- `cargo publish --dry-run`
